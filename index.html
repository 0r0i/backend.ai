Architecture

Sorna consists of three loosely coupled components.

  • sorna-gateway: Runs an HTTP REST API server for public access
  • sorna-manager: Maintains the instance registry and distributes user requests
  • sorna-agent: Manages each instance and runs user-input codes inside Docker containers

We call each code-running container a kernel. All kernels have our custom-built sandbox (called "sorna-jail") that secures our infrastructure on the system-call level. The Dockerfile recipe and REPL handlers for each kernel type is available via sorna-repl subproject.

Additionally, we provide a pluggable sorna-media Javascript library and Python packages for front-end services to render interactive graphics and handle multi-media outputs generated from kernels.
